## Status Key

| Status | Meaning |
|--------|---------|
| ❌ Needs Plan | Gap identified, no plan yet |
| 📋 Planned | Ready to implement |
| 🚧 In Progress | Being worked on |
| ⏸️ Blocked | Waiting on dependency |
| ✅ Complete | Implemented and verified |

## Creating a New Plan

1. Copy `TEMPLATE.md` to `NN_name.md`
2. Fill in gap, steps, acceptance criteria, and verification
3. Add the plan to this index
4. Commit with `[Plan N]` or `[Plan #N]` prefix

## Trivial Changes

Not everything needs a plan. Use `[Trivial]` for:
- less than 20 lines changed
- no new files created

## Completing Plans

Update the plan status and verification evidence in the plan file before closeout.
